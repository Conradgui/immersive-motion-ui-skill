# Brand UI

Load this for brand landing pages, launch pages, portfolios, agencies, campaigns, editorial pages, and cinematic heroes.

## Job

A brand page must make the subject memorable while still making the next action obvious. It is not a poster. It must survive responsive layout, real copy, and browser verification.

## Structure

A strong brand page usually has:

1. First viewport signal: the brand, product, place, or person is immediately visible.
2. Subject-shaped composition: layout, type, material, and motion come from the brief.
3. One signature move: a single recognizable visual or interaction idea.
4. Next-section hint: the first viewport should reveal that the page continues.
5. Real action path: CTA, navigation, or inquiry path is usable.

## First Viewport

Do:

- make the subject visible, not only named in tiny nav text
- use actual product/place/person/media when available
- let the next section peek into view on desktop and mobile
- keep H1 literal: product, brand, person, place, or offer category

Avoid:

- split hero with text card on one side and generic visual on the other
- decorative gradient-only hero
- dark stock-like blur that hides the subject
- giant marketing copy that says nothing specific
- full-screen intro that blocks content without purpose

## Motion

Motion should create memory, reveal hierarchy, or make transitions legible.

Use:

- one scroll or entrance system
- stagger only when it clarifies reading order
- reduced-motion fallback for continuous or large motion
- media motion when it directly supports subject understanding

Avoid:

- multiple competing animation systems
- motion that delays primary content
- parallax that hides text or breaks mobile reading

## Visual Quality Checks

Before finalizing, inspect:

- Does it look like this exact subject, or could it fit any startup?
- Is the signature move visible in the first 5 seconds?
- Does typography have hierarchy beyond size alone?
- Are colors more than one hue family with tiny variations?
- Does the first viewport avoid becoming a closed poster?
- Can the user still find the next action?

## Common Fixes

| Symptom | Fix |
| --- | --- |
| too generic | add subject-specific material, interaction, or image logic |
| too empty | add structured proof, detail band, or next-section hint |
| too loud | reduce competing effects and keep one signature move |
| too flat | add typographic contrast, layered media, or spatial rhythm |
| too much like SaaS | remove generic cards and use editorial/product-specific composition |
