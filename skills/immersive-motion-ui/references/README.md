# Core References

These references support the Codex-only Core Skill. Load them selectively through the routing table in `SKILL.md`; do not load the full directory by default.

They are the standalone domain knowledge of Core, not placeholders for Library content. Each active module must remain actionable when the optional Library is missing.

Available now:

1. `routing.md` - choose register, command, and reference path.
2. `design-dna.md` - avoid generic AI defaults and choose a subject-specific design move.
3. `visual-directions.md` - calibrate creative ambition and adapt a subject-specific direction prototype.
4. `brand-ui.md` - build premium brand, portfolio, launch, campaign, and editorial pages.
5. `product-ui.md` - build dense, calm dashboards, admin tools, analytics UIs, and app shells.
6. `anti-cheap.md` - audit generic, fragile, or dishonest UI output.
7. `preflight.md` - inspect project stack, edit boundary, and verification path before implementation.
8. `redesign.md` - safely improve existing UI while protecting working structure and brand assets.
9. `commerce-ui.md` - build PDP, PLP, cart, checkout, pricing, booking surfaces, and honest reservation lifecycle states.
10. `motion-engines.md` - plan purposeful motion, cinematic moments, and reduced-motion fallbacks.
11. `media-assets.md` - choose, source, generate, integrate, and verify imagery or media without requiring Library assets.
12. `image-generation.md` - route among existing assets, Library candidates, ImageGen, authentic sources, and no-media fallbacks.
13. `verification-evidence.md` - report build, static audit, browser, viewport, console, overflow, and reduced-motion evidence honestly.
14. `browser-debugging.md` - probe optional browser tooling, inspect safely, preserve write boundaries, and capture reproducible runtime evidence.
15. `workflow-state.md` - keep `design-model.yaml` and `worklog.md` for long-running UI work.
16. `library-integration.md` - discover the optional MotionSites Library after Core routing and degrade gracefully when absent.
17. `dependency-policy.md` - classify conditional tools and services, probe availability, use fallbacks, and prevent unsupported claims.
18. `industry-directions.md` - choose among six task-first industry families without hard-mapping sectors to a fixed visual style.
19. `showcase-casebook.md` - route to one of six locked-fixture before/after cases without loading the full case set.
20. `performance.md` - diagnose LCP with a four-part trace model and report static risk separately from measurement.
21. `memory-lifecycle.md` - assign cleanup ownership and compare baseline, target, and final snapshots without reading raw heaps.
22. `a11y-audit.md` - combine automated audits with accessibility tree, DOM order, labels, keyboard, focus, targets, and contrast evidence.
23. `data-ui.md` - design high-density analytical surfaces with honest units, sources, states, and table fallbacks.
24. `data-apps.md` - select an existing data-app stack without assuming a chat, cloud, or visualization dependency.
25. `design-system.md` - maintain primitive and semantic tokens, modes, variants, and component states across pages.
26. `eval-protocol.md` - freeze isolated comparison tracks and make only evidence-supported quality claims.

Do not place the full MotionSites prompt corpus here. Prompt templates, media, demos, and gallery assets belong in the optional Library repo.

Implementation examples live in `../examples/`: three offline Core pairs plus five explicitly bounded Finesse network/source references. Load them selectively; they are not default templates.
