# Design DNA

Load this when choosing the visual direction, fighting generic output, or interpreting feedback such as "premium", "more soul", "too plain", or "too AI".

## Core Rule

Premium UI is not a palette. It is a clear relationship between subject, audience, job, materials, density, motion, and evidence.

Before styling, answer:

1. What is the page or app trying to make the user do?
2. What would the category default look like?
3. Why is that default wrong or insufficient here?
4. What is the one signature move that belongs to this subject?

## Anti-Default Check

Avoid these unless the brief truly calls for them:

- centered hero, soft gradient, rounded cards, generic SaaS copy
- dark blurred background with tiny bright text
- purple/blue gradient as the main idea
- oversized decorative cards in operational tools
- beige editorial layout for every luxury or culture topic
- pure black/white contrast with one neon accent
- placeholder names, fake metrics, and vague claims

Do not ban a pattern forever. A pattern is wrong only when it is unearned, generic, or harmful to the job.

## Signature Move Menu

Choose one primary move. More than one can make the UI feel unfocused.

| Move | Use When | Risk |
| --- | --- | --- |
| editorial scale shift | brand, portfolio, campaign | can become magazine template |
| dense control surface | product/admin/analytics | can become cramped |
| cinematic media plane | launch, place, product story | can hide content |
| tactile material system | luxury, hardware, craft, spatial UI | can become decoration |
| data-as-composition | analytics, finance, science | can reduce readability |
| strict typographic system | professional, institutional, enterprise | can feel cold |
| asymmetric navigation | portfolio, agency, editorial | can harm orientation |

## Product Meaning

Design direction is a product decision:

- A brand page should be memorable enough to be recognized later.
- A product UI should reduce repeated cognitive work.
- Commerce should make inspection, comparison, trust, and action easier.
- A redesign should improve the requested dimension without erasing what already works.

## Copy Rules

Use concrete, domain-shaped language.

Avoid:

- "Elevate"
- "Seamless"
- "Unleash"
- "Cutting-edge"
- fake round metrics
- "John Doe"
- filler labels such as "Solutions", "Innovation", "Experience" with no object

Prefer:

- specific nouns
- actual user actions
- measurable constraints when real
- labels that describe what the interface controls or reveals
