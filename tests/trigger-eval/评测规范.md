# Core 触发代理评测规范 v1

日期：2026-07-13
状态：运行前冻结

## 目标

评测独立代理仅依据可见 Skill 能力描述，是否应激活 immersive-motion-ui 的分类准确性。它是 Codex 自动触发的可复现实验代理，不是平台线上遥测。

## 标签

| 标签 | 定义 | 计入激活正例 |
| --- | --- | --- |
| primary | Core 应主导该前端设计/实现工作 | 是 |
| with-specialist | Core 应参与 UI 判断或验证，同时需要浏览器、性能、内存或 a11y 专项路径 | 是 |
| out-of-scope | 任务没有实质前端设计、UI 实现或 UI 验证职责 | 否 |

标签判断看任务的主要交付，不因为任务出现 React、网页或用户就自动触发。纯后端、数据库、云、Extension、文案、图片生成、DevOps、静态数据处理和安全任务属于负例，除非题干明确要求 UI 工作。

## 数据集

- development-v1：36 条，供一次描述调整前的诊断使用。
- holdout-v1：36 条，独立验收；不得用于改写描述、词表、权重或标签。
- 每份：primary 12、with-specialist 8、out-of-scope 16；中英文各 18。
- 题干不得点名本 Skill、不得泄露标签、不得要求调用某个 Skill。

## 盲测

每位独立 Subagent 仅获得任务 id 与 prompt、三类标签定义和严格 JSON 输出格式；不得读取仓库、fixtures、金标、评分阈值或既有结果。两位评测者一次性完成全部 72 条，评分在两份预测都固化后才进行。

## 指标

1. activation Precision / Recall / F1：primary 与 with-specialist 合并为应激活。
2. role Macro-F1：分别衡量三类标签，防止只会粗暴触发。
3. critical false positive rate：标记为高风险负例的任务被激活的比例。
4. 评测者一致性：两位 Subagent 的三类标签 exact agreement。

## 目标门槛

- 预测记录完整率：100%。
- activation Precision >= 0.90。
- activation Recall >= 0.85。
- activation F1 >= 0.87。
- role Macro-F1 >= 0.80。
- critical false positive rate = 0。

每位评测者单独报告，不用平均分掩盖其中一位明显失准。通过说明独立代理触发代理评测通过；不说明 Codex 平台线上自动触发已被证明。

## 调整规则与停止条件

首次运行后只允许基于 development-v1 对 SKILL.md description 做一次小范围调整。holdout-v1 始终不参与调参。若首次全量评测已达标，本批停止；若未达标，记录失败模式后单开一次 description 调整批次。
